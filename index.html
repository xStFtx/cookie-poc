MGM API Vulnerability

Proof of Concept for Bug Bounty Submission

IMPORTANT: This proof of concept is intended for legitimate security research and bug bounty reporting only.

Vulnerability Overview

The MGM Grand Paradise Macau booking API (booking.mgm.mo) has an endpoint that:

  • Accepts arbitrary user data without authentication (email, phone, PMS ID)
  • Does not implement CSRF protection (no CSRF token required)
  • Does not validate the origin of requests (accepts cross-origin requests)
  • Exposes sensitive data in responses based on provided parameters

This vulnerability allows potential attackers to:

  • Make API calls using an authenticated user's cookies
  • Submit arbitrary data to MGM's systems
  • Potentially access or manipulate reservation data

API Vulnerability Demonstration

Vulnerable Endpoint

POST /api/Reservation/ByPmsid/get HTTP/2 Host: booking.mgm.mo

Request Parameters

The API accepts the following JSON parameters:

{ "lang": "en-US", "hotelCode": "999", "areaCode": "1", "currencyCode": "USD", "phone": "12", "email": "admin@gmail.com", "pmsid": "11251352" }

Vulnerability Analysis:

Cross-Site Request Forgery (CSRF) Proof

The API endpoint is vulnerable to CSRF attacks because:

  • It does not require a CSRF token
  • It accepts and processes requests with user cookies
  • The endpoint can be called from any origin

Real-World Impact & Recommendations

                
                

                    
Security Impact

                    
This vulnerability could potentially allow attackers to:

                    
    
                        
  • Access reservation details using brute force PMS ID attempts
    • 
                        
  • Submit fraudulent information to MGM's booking systems
    • 
                        
  • Perform actions on behalf of logged-in users via CSRF attacks
    • 
                        
  • Potentially access PII through crafted API requests
    • 
                    

                

                
                

                    
Recommendations

                    
    
                        
  • Implement proper authentication for all API endpoints
    • 
                        
  • Add CSRF token validation for state-changing requests
    • 
                        
  • Validate request origin and implement proper CORS policies
    • 
                        
  • Rate limit API requests to prevent brute force attacks
    • 
                        
  • Implement proper input validation for all parameters
    • 
                        
  • Add server-side validation to verify user authorization for accessing reservation data
    •
